# 玩家死亡后滑移问题分析与修复方案

## 问题背景

在 `LevelScene` 与 `BossScene` 中，玩家死亡后会沿死亡前的速度方向持续移动，直到被场景边界阻挡，导致“尸体滑行”现象，影响战斗观感与一致性。

## 现象与复现

### 现象
- 玩家被小怪或 Boss 击杀后，死亡文字已出现，但角色仍继续位移 1-2 秒。
- 位移方向通常与死亡瞬间移动或闪避方向一致。

### 复现步骤
1. 进入任一战斗场景（普通关卡或 Boss 战）。
2. 保持移动或闪避状态下被击杀。
3. 观察死亡文字出现后，玩家位置仍持续变化。

## 根因分析

问题由以下两点叠加触发：

1. **死亡触发时未立即清零速度**
   - 场景内进入 `playerDead` 分支后，仅设置死亡状态和延迟返回 Hub。
   - 未在死亡瞬间执行 `setVelocity(0, 0)` 或等价冻结逻辑。

2. **场景 update 提前 return**
   - `playerDead = true` 后，后续帧在场景 `update` 早退，不再进入玩家更新流程。
   - 物理系统沿用死亡前一帧速度继续推进角色，形成滑移。

## 修复方案

### 方案 A（最小修复）
- 在 `LevelScene` 与 `BossScene` 的死亡触发处直接增加：
  - `player.setVelocity(0, 0)`
  - `player.body.setVelocity(0, 0)`（若 body 存在）
  - `player.isDodging = false`
  - `player.isAttacking = false`

**优点**：改动小、见效快。  
**缺点**：死亡冻结逻辑分散在多个场景，后续易漏改。

### 方案 B（推荐）
- 在 `Player` 类新增统一方法（例如 `handleDeathFreeze()`）：
  - 统一执行速度清零、动作状态清理、可选碰撞体禁用。
- `LevelScene` 与 `BossScene` 死亡分支统一调用该方法。

**优点**：职责集中、可复用、后续维护成本低。  
**缺点**：较方案 A 多一层重构。

### 方案 C（演出冻结）
- 死亡后短时间 `physics.world.pause()` 冻结全场，过场后恢复/切场。

**优点**：绝对避免滑移并强化死亡演出。  
**缺点**：会冻结敌人与特效，节奏变化较大。

## 推荐结论

推荐采用 **方案 B**（统一封装 + 场景接入），在保证修复效果的同时降低未来同类问题复发概率。

## 验收标准

- 玩家在 `LevelScene` 死亡后下一帧位置不再变化。
- 玩家在 `BossScene` 死亡后下一帧位置不再变化。
- 死亡后 2 秒返回 Hub 的流程保持不变。
- 不引入新的输入卡死或异常动画状态。
