# 像素瓦片地板差异化设计

## 目标

将当前 `HubScene`、`LevelScene`、`BossScene` 的纯色地板改为像素瓦片风格，并保证不同地图拥有可辨识的地板差异（配色与图案噪声不同）。

## 现状

- 目前地板由 `Graphics.fillRect` 直接绘制纯色矩形。
- Hub 使用统一网格线；Level/Boss 使用区域矩形 + 边框。
- 场景逻辑（可行走区域、碰撞、刷怪）独立于地板贴图，视觉替换风险较低。

## 方案结论

采用**程序化生成像素瓦片纹理**并平铺到地板区域：

- 在运行时生成多个瓦片纹理（无需新增美术素材）。
- 每个地图使用独立风格配置（主色、阴影、亮点、噪声比例、条纹比例）。
- 保持所有地图原有边界与可行走区域逻辑不变，仅替换视觉层。

## 设计细节

### 1) 纹理生成

- 新增通用函数，按配置生成 `tileSize x tileSize` 像素纹理。
- 每个像素点根据稳定伪随机与阈值选择 `base / shade / highlight / accent`。
- 生成命名约定：`pixel_tile_<styleKey>`。

### 2) 地板平铺

- 新增通用函数，按矩形区域用 `Image` 重复平铺目标纹理。
- 默认深度 `0`，与当前地板层级一致。
- 支持可选描边边框颜色，保留关卡和 Boss 房间视觉边界。

### 3) 地图差异化策略

- Hub：偏中性石板色，低对比度，轻微条纹。
- 七宗罪地图：按 `bossKey` 配置对应色相（如 wrath 偏熔岩红、envy 偏毒沼绿）。
- Boss 房间：沿用对应 `bossKey` 风格，但提高对比度与边框强调。

## 风险与回退

- 风险：瓦片太花导致角色可读性下降。
- 对策：控制亮点比例与对比度，必要时统一降低饱和度。
- 回退：只需切回原 `fillRect` 绘制分支。

## 验收标准

- Hub、Level、Boss 三类场景均为像素瓦片地板。
- 不同地图地板在颜色和纹理上明显可区分。
- 玩家移动、碰撞、战斗与过场不受影响。
