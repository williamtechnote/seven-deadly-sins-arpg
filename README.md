# 七宗罪 | Seven Deadly Sins

基于 Phaser 3 的浏览器像素风动作 RPG。操控主角挑战以七宗罪为主题的七大 Boss，通过多武器切换、闪避翻滚和策略搭配征服每一场战斗。

## 快速开始

无需安装依赖，启动本地 HTTP 服务器即可运行：

```bash
# Python
python3 -m http.server 8000

# 或 Node.js
npx serve .
```

浏览器打开 `http://localhost:8000`。

## 操作指南

| 按键 | 操作 |
|------|------|
| WASD | 八方向移动 |
| J / 鼠标左键 | 普通攻击 |
| K / 鼠标右键 | 特殊攻击 |
| Space | 闪避翻滚（无敌帧） |
| Q / E | 切换武器 |
| 1-4 | 使用快捷栏道具 |
| Tab | 背包 |
| H | 操作指引 |
| Esc | 暂停菜单（继续/背包/武器信息/主静音/主音量/返回标题） |
| F | NPC 交互 |

> 游戏内随时按 **H** 可查看完整操作指引。

### Hub — 净罪庇护所

安全区域，包含：
- **铁匠** — 升级武器 + 制作实用药剂（净化药剂/狂战油）

- **Hub 小地图** — 显示房间、传送门、NPC 和玩家位置

## 新增机制

- **战斗基线重调**：Boss/小怪生命值恢复正常，Boss 多阶段机制可触发。
- **经济闭环**：普通小怪可掉落金币、药水、低概率区域精华；掉落需拾取并进入背包，有浮字反馈。
- **Debuff 生效**：`reverseControl` 会在命中后短时间反转移动控制。
- **对话立绘映射**：优先使用 NPC/Boss 对应精灵头像，无法匹配时自动降级为占位色块。
- **最小音频系统**：内置 WebAudio 程序化音效（攻击/受击/UI）与可选环境循环，无需外部音频资源。
- **状态效果系统（Phase 1）**：新增 `burn / bleed / slow`，具备持续时间、平衡数值、浮字与光环反馈；已接入多把武器特攻与敌方/Boss 技能。
- **Run Modifiers（Phase 1）**：每局随机抽取 3 个词缀（HUD 与 H 帮助页显示），影响敌人速度/血量、玩家伤害与承伤、掉落率等。
- **锻造制作（Phase 1）**：铁匠新增 2 个实用消耗品配方（净化药剂、狂战油），消耗金币与精华，配方固定且可回归验证。
- **Boss 可读性（Phase 1）**：阶段切换会显示文本提示 + 反制建议；阶段关键新招式首次施放前增加短预警延迟，并新增“反制窗口秒数”提示，提升可操作性。
- **可复玩目标（Phase 1）**：每局随机生成挑战目标（当前为击败敌人数挑战），HUD 实时显示进度并在完成时发放金币奖励。
- **事件房基础框架（Phase 1）**：每局生成一个事件房原型（名称/描述/状态），在 HUD 显示“未发现/已发现/已触发”，为后续事件房玩法扩展打底。
- **战斗反馈增强（Phase 1）**：玩家命中敌人/Boss 时新增瞬时冲击环视觉反馈；特攻命中会触发更强冲击环与“重击/破势”浮字，并显示伤害数字；玩家受击时也会显示红色伤害数字，且低血量（≤30%）与低体力（≤20%）出现闪烁警示（含实时百分比）；血条/体力条与对应数值文本都会随风险分级变色，提升双向反馈清晰度。
- **战斗反馈增强（Phase 2）**：快捷栏治疗/体力消耗品现在会显示实际恢复量，资源已满时不会白白消耗；底部状态 HUD 拆分为“负面/增益”双行，并按剩余时间优先排序，更容易临场读状态。

## 游戏流程

```
主菜单 → 净罪庇护所 → 选择传送门 → 关卡探索（小怪战斗）
→ Boss 战 → 剧情对话 → 获得奖励 → 返回庇护所
→ 升级武器 / 补给 → 下一个传送门 → ……
→ 集齐七枚罪之印记 → 最终 Boss → 通关
```

## CLI 回归检查

项目包含轻量级功能回归脚本（不影响浏览器运行时）：

```bash
node scripts/regression-checks.mjs
```

覆盖项：
- 武器属性成长单调性（强化后伤害不降、攻速/冷却/体力消耗不升）
- 武器绑定材料校验与扣除逻辑
- 存档序列化/反序列化完整性
- 状态效果配置与 tick 伤害逻辑
- 消耗品实际恢复量结算与“已满不消耗”规则
- HUD 状态分组与倒计时排序稳定性
- Run Modifier 抽取唯一性与效果合成
- 铁匠制作配方消耗与产出确定性

## 自动持续迭代（每 20 分钟）

已提供可周期运行的本地机制，通过 `launchd` 每 20 分钟触发一次 Codex 自动研发循环，并通过 `TODO.md / PROGRESS.log / README.md` 传递上下文。

### 安装并启动

```bash
./scripts/install-auto-iteration.sh
```

### 查看状态与日志

```bash
launchctl print gui/$(id -u)/com.william.sevensins.autocycle

# 调度层日志
ls -lah .auto-iteration/
# 主要看：
# - .auto-iteration/runner.log
# - .auto-iteration/launchd.out.log
# - .auto-iteration/launchd.err.log
```

### 停止并卸载

```bash
./scripts/uninstall-auto-iteration.sh
```

### 机制说明

- 调度间隔：`1200s`（20 分钟）
- 防重入：`scripts/auto-iterate.sh` 使用锁目录避免并发执行
- 每轮要求：
  - 读取 `TODO.md / PROGRESS.log / README.md`
  - 选取最早 2 个 active 项进行实现
  - 分支命名固定为 `feat/auto-[short-description]`
  - 跑回归测试并执行 commit/merge/push 规则
  - 强制写入 `PROGRESS.log` 审计记录
- 分支保留策略：不删除 feature 分支（本地与远端均保留）
