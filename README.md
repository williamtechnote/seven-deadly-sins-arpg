# 七宗罪 | Seven Deadly Sins

基于 Phaser 3 的浏览器像素风动作 RPG。操控主角挑战以七宗罪为主题的七大 Boss，通过多武器切换、闪避翻滚和策略搭配征服每一场战斗。

## 快速开始

无需安装依赖，启动本地 HTTP 服务器即可运行：

```bash
# Python
python3 -m http.server 8000

# 或 Node.js
npx serve .
```

浏览器打开 `http://localhost:8000`。

## 操作指南

| 按键 | 操作 |
|------|------|

`本局挑战：` / `挑战：` 前缀，compact 第二行也会先去重再拼接奖励短句，避免紧凑摘要重复“挑战”标题；当 compact 第二行的宽度预算继续吃紧时，进行中与完成态都会先沿用 `击败 30 个敌人 · +90金 -> 击败 30 个敌人 -> 击败30个敌人` 这条语义回退链，再退化到通用省略；若当前 challenge 没有奖励短句，则 compact 第二行会继续沿用 `击败 30 个敌人 -> 击败30个敌人` 这条 label-only 回退梯子，不补 `+0金` / `奖励:未知` 这类占位；若进一步进入 ultra-compact 档位，则会先收紧区块间距与底边缓冲，本局词缀会压到 1 行、本局挑战会压到单行 `挑战 进度 · 奖励` 摘要；当这条可见摘要的宽度预算继续吃紧时，即使奖励数值扩大到 `+9999金` 这类长度，进行中态也会继续沿用 `挑战 12/30 · +90金 -> 挑战 12/30 -> 12/30` 这条语义回退链，完成态则继续沿用 `挑战完成 · +90金 -> 挑战完成 -> 完成`，而不会额外插入新的中间短句；若当前 challenge 没有奖励短句，则 ultra-compact 单行摘要也会继续沿用 `挑战 12/30 -> 12/30` / `挑战完成 -> 完成` 这条 no-reward 回退梯子，不补 `+0金` / `奖励:未知` 这类占位；若未来扩展到 `+9999金 +净化` 这类复合奖励短句，也会继续沿用同一条可见摘要与完成徽记回退链；regular / compact 分档里凡是仍会显示奖励的路径，也会复用同一奖励短句 helper，避免与 ultra-compact 回退链出现文案漂移；共享 challenge 标签与显式奖励短句 helper 也会压缩异常半角 / 全角空白，避免正文间距或复合奖励文案因脏输入而提前挤爆各分档宽度预算；若该挑战摘要与本局词缀正文都因溢出被隐藏，则会在挑战起步后把 `进12/30` / `完成` 这类更轻量的进度徽记挂到“本局词缀”标题后；若标题预算进一步吃紧，则进行中态还会继续压成 `12/30`；若进入 ultra-tight 更紧预算，则会再回退为 `进12` 这类无省略最终短句；若连进行中态的 `进12` 都放不下，则也会静默隐藏 badge，把同一行预算完全还给标题；完成态还会先从 `完成+90金` 这类奖励短句回退为 `完成`；若连完成态的 `完成` 都放不下，则会静默隐藏 badge，把同一行预算完全还给标题；该轻量徽记会拆成独立弱化色阶，并进一步下调字级与透明度后再与“本局词缀”标题分开贴边；若标题预算继续压窄，则会按更紧预算分档继续下调 badge 宽度占比、最小宽度与固定 gap，优先把更多横向空间留给标题正文；若侧栏仍超出安全高度，则挑战摘要会排在事件房摘要与本局词缀正文之后，作为最后一级隐藏项。
- **事件房扩展（Phase 2）**：每局会随机生成 1 个事件房并放入关卡。进入事件点后按 `F` 打开抉择面板，再用 `1/2` 选择结算路线；结果会同步写入 HUD 与存档。当前抉择面板会直接显示与 HUD 对齐的紧凑路线预览短句，并在生命变化路线旁追加 `预估生命+/-N`，在金币消费路线旁追加 `可负担 / 金币不足` 短标签；当玩家靠近未结算祭坛时，世界里的靠近提示也会直接切成 `按F交易` / `按F治疗` / `按F效果` 这类房型短标签，未知或未来房型则继续回退为通用 `按F抉择`。事件房祭坛靠近提示也会按 Phaser 文本实际宽度贴在当前视口内，因此贴近屏幕边缘时不会被裁出画面；已触发后的祭坛世界标签也会复用同一套实际测量与省略钳制，因此 `房名 · 已选: ...` 这类长摘要贴近镜头边缘时会优先留在视口内，而不是继续横向溢出。若路线因金币不足、已结算或状态异常而无法执行，面板会保留并在底部给出明确阻塞提示，而不是静默关闭。右上 HUD 会额外显示压缩后的事件房类型/状态标签，并把未结算路线拆成逐行摘要；已结算事件房则会把路线与结算合并成单行摘要，其中 `风险增益 / 祝福` 类会显示 `效果:` 前缀，`交易` 类会显示 `交易:` 前缀，`治疗` 类会显示 `治疗:` 前缀；右侧固定侧栏里的章节标题、区域名、本局词缀、本局挑战与事件房摘要会优先按 Phaser 文本实际宽度钳制，并按实际文本高度动态纵向排布，因此长标题 / 长区域名 / 长词缀名 / 长路线摘要 / 长结算说明会先在 HUD 内收紧并顺序下推，而不是继续只靠静态换行；若视口进入 compact 档位，则本局词缀与事件房摘要会额外收敛为有限行数，并在最后一行补省略号，本局挑战摘要会压缩为 2 行紧凑版；若进一步进入 ultra-compact 档位，则会先进一步收紧各区块间距与底边缓冲，本局词缀会压到 1 行、事件房摘要压到 2 行，本局挑战会压到单行 `挑战 进度 · 奖励` 摘要；若侧栏总高度仍超出安全范围，则会优先隐藏事件房摘要，其次再隐藏本局词缀正文，最后才隐藏本局挑战摘要。场景里的祭坛世界标签在已结算后也会直接追加同一条类型化路线短摘要，便于回头路快速辨认。若已知房型的已触发存档缺少选项标签或结算文本，也会继续保留各自的房型前缀，并回退为 `未知选项` / `结算待同步`，避免摘要退化为无前缀空洞文案；其中 `疗愈泉眼` 在选项标签与结算文本同时缺失时，也会继续保留完整双回退 `治疗: 未知选项 · 结算待同步`，而不是回填当前事件池里的标签；若未来/未知房型的已触发选项已脱离当前事件池定义，HUD 会优先沿用存档里的选项标签，并回退为通用 `已选:` 前缀；对应的祭坛世界标签也会继续沿用同一条 `已选:` 路线前缀，若存档路线标签缺失则补成 `已选: 未知选项`，避免未来扩展后回头路标签失去辨识度；若 HUD 侧该标签缺失但仍保留结算文本，则会补上 `未知选项` 占位并继续输出 `已选: 未知选项 · <结算>`；若该存档路径只缺结算文本，则会补上 `结算待同步` 占位文案；若两者同时缺失，则继续保留完整双回退 `已选: 未知选项 · 结算待同步`，避免已触发摘要退化成空行。若未来扩展后连事件定义本身都已移除，祭坛世界标签仍会沿用存档房名并回退为 `房名 · 已结算`，避免地图上的已触发事件点变成空白标签。当前已实现 5 个原型：`赌徒圣坛`（生命换金币）、`疗愈泉眼`（高额治疗或治疗+净化负面状态）、`血契祭坛`（本局增伤但提高承伤，分激进/稳健两档）、`战备商柜`（花金币换 `净化药剂 / 狂战油` 直接入包）、`祈愿圣坛`（在体力恢复强化与特攻冷却缩短之间二选一）。
- **战斗反馈增强（Phase 1）**：玩家命中敌人/Boss 时新增瞬时冲击环视觉反馈；特攻命中会触发更强冲击环与“重击/破势”浮字，并显示伤害数字；玩家受击时也会显示红色伤害数字，且低血量（≤30%）与低体力（≤20%）出现闪烁警示（含实时百分比）；血条/体力条与对应数值文本都会随风险分级变色，提升双向反馈清晰度。
- **战斗反馈增强（Phase 2）**：快捷栏治疗/体力消耗品现在会显示实际恢复量，资源已满时不会白白消耗；底部状态 HUD 拆分为“负面/增益”双行，并按剩余时间优先排序，更容易临场读状态。
- **Boss HUD 可读性增强（Phase 3）**：Boss 血条现在带受击残影、转阶段警戒闪烁、`Phase x/y` 标签与后续阶段阈值刻度；关键大招预警条会贴近血条显示攻击类型、反制窗口与提示，并会按 Phaser 文本实际测量压缩长攻击名 / 长反制提示，避免挤出顶部 HUD；当 Boss 被特攻打断进入短暂`破招`，或处于减速等受控状态时，血条上会叠加高亮段，能更直观地判断压血进度、读招窗口与反打时机。

## 游戏流程

```
主菜单 → 净罪庇护所 → 选择传送门 → 关卡探索（小怪战斗）
→ Boss 战 → 剧情对话 → 获得奖励 → 返回庇护所
→ 升级武器 / 补给 → 下一个传送门 → ……
→ 集齐七枚罪之印记 → 最终 Boss → 通关
```

## CLI 回归检查

项目包含轻量级功能回归脚本（不影响浏览器运行时）：

```bash
node scripts/regression-checks.mjs
```

覆盖项：
- 武器属性成长单调性（强化后伤害不降、攻速/冷却/体力消耗不升）
- 初始长剑攻击范围基线，以及玩家死亡冻结 hook 在普通关卡 / Boss 战的接入
- 普通小怪基础移速下调基线，以及长剑对普通小怪/Boss 的开场安全先手窗口守卫
- 武器绑定材料校验与扣除逻辑
- 存档序列化/反序列化完整性
- 状态效果配置与 tick 伤害逻辑
- 消耗品实际恢复量结算与“已满不消耗”规则
- HUD 状态分组与倒计时排序稳定性
- Boss 血条残影推进、阶段 HUD 摘要、关键大招预警条与破招/受控高亮段
- 键盘双轴瞄准 helper、`当前瞄准` HUD 提示、`普攻 U / 特攻 O` 冷却状态行、紧凑快捷栏标签、背包点击自动装入快捷栏首个空位时的 `快捷栏N：+<短名>` 文案、缺少显式短名时沿用道具名词干生成的 `快捷栏N：+生命` 这类回退短句、超长词干截成 `快捷栏N：+圣疗秘…` 这类省略短句、满快捷栏时回写 1 号槽位的 `快捷栏1：<旧短名>→<新短名>` 提示、同短名覆盖时压缩为 `快捷栏1：同类 <短名>`、缺少显式短名时改用 `快捷栏1：狂战→净化` 这类道具名短句，以及超长覆盖标签截成 `快捷栏1：古代狂…→神圣净…` 的规则，还有 `Tab -> 点击背包消耗品 -> 1-4 使用` / `IJKL + U/O + Q/E + Space` 键盘战斗循环的 source hooks
- 右侧固定侧栏的实际显示尺寸优先分档守卫，以及由同一 display-size 响应式预算推导出的 challenge badge `maxWidth / gap` 分档联动、进行中/完成态 badge 在静默隐藏时的外观清空、`_updateRunModifierHeading` 空 badge 路径对样式与标题预算的同步复位，避免逻辑视口与真实显示尺寸脱钩或旧 tint 残留后再次回到不可触发/误着色状态
- 事件房 HUD 压缩类型/状态标签、逐行路线摘要、祭坛世界标签的已选路线短摘要、靠近祭坛时的 `按F交易` / `按F治疗` / `按F效果` 提示，以及已触发后合并的“已选路线 + 结算”单行短句（含未知/未来房型靠近提示的 `按F抉择` 回退、`疗愈泉眼` 的 `治疗: 未知选项 · 结算待同步` 回退、未知房型在 HUD/祭坛标签上的 `已选:` / `已选: 未知选项` / `已选: 未知选项 · 结算待同步` 回退，以及事件定义缺失时祭坛标签的 `房名 · 已结算` 回退）
- 事件房抉择元数据持久化，以及 `赌徒圣坛 / 疗愈泉眼 / 血契祭坛 / 战备商柜 / 祈愿圣坛` 的结算、本局效果逻辑与抉择面板里的生命预估 / 金币可负担短标签
- Run Modifier 抽取唯一性与效果合成
- 铁匠制作配方消耗与产出确定性
